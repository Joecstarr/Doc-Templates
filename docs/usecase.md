> [!note]
>
> The status indicates the priority of the use case to the stakeholders one of:
>
> - Low: The use case would be useful but everything else should be completed before it is realized.
> - Medium: The use case is needed for the product but not a showstopper. The use case should be
>    realized when practical.
> - High: The use case is important for the product, missing this use case means the product is
>   incomplete. The use case must be realized.
> - Critical: The use case is so important that missing it means the product isn't just
>   incomplete it's straight up unusable. The use case should be realized before anything else.
